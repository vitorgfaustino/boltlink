2. Finalidade

Este serviço é utilizado para criar, administrar e redirecionar links curtos, com contagem agregada de acessos e proteção administrativa.

3. Dados persistidos pelo produto

  • slug do link
  • URL de destino
  • contador agregado clicks_total
  • datas operacionais, tags, grupos e QR code
  • hash de senha, quando o link for protegido

4. Dados que não são persistidos por padrão

  • IP em texto puro
  • hash de IP
  • país por clique
  • Referer
  • User-Agent
  • eventos individuais por clique

5. Referrer e redirecionamento

Nos redirects públicos, esta instância utiliza Referrer-Policy: strict-origin. Isso permite que o destino receba apenas a origem do tráfego quando o navegador decidir enviá-la, sem encaminhar o caminho completo nem a query string como referrer.

6. Segurança

O painel administrativo e a API de gestão podem ser protegidos por Cloudflare Access e por controles adicionais definidos pelo operador do ambiente.

7. Infraestrutura e provedores

Esta instância pode utilizar serviços de infraestrutura e segurança de terceiros, como Cloudflare, conforme a configuração feita pelo operador. O operador deve listar aqui os subprocessadores efetivamente usados.

8. Direitos do titular

O operador desta instância deve informar aqui os canais para exercício de direitos previstos na legislação aplicável, incluindo acesso, correção, eliminação, bloqueio e informações sobre compartilhamento, quando cabíveis.

9. Responsabilidade pelo uso desta instância

O software BoltLink é distribuído sob AGPL-3.0 como software livre. O autor original do software não se torna automaticamente controlador, operador ou encarregado dos dados tratados por quem implanta esta instância.

A responsabilidade pela operação concreta deste ambiente, pela definição de finalidade, base legal, retenção, conteúdo inserido e serviços conectados é exclusiva do operador desta instância.
